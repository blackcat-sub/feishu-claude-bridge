# 用飞书在手机上遥控 Claude Code 干活，支持语音和图片

**适合谁：用 DeepSeek 等纯文本模型、不想为多模态额外付费、想在手机上指挥电脑干活的飞书用户。**

手机飞书发文字、语音或图片 → Claude Code 电脑端自动处理 → 飞书回复结果。

核心优势：
1. **零服务器** — 不需要服务器、Webhook、数据库，全在你电脑上跑
2. **无需多模态模型** — 纯文本模型（DeepSeek 等）也能处理语音和图片，不额外花钱
3. **隐私在你手里** — 语音走飞书自己的妙记（没引入第三方 ASR），图片走本地 tesseract（没调云 API）。飞书知道你的消息这事你本来就没法避免，但我们没有在此基础上再增加数据出口
4. **全本土生态** — 飞书 + DeepSeek 都在国内，普通网络就能稳定运行。不需要特殊网络环境，不依赖任何境外服务

## 使用场景与局限

这个项目最适合处理**以文字信息为主**的图片：聊天截图、文档拍照、表格、PPT 截图等等。绝大部分飞书使用场景里，发图片就是为了传递上面的文字，OCR 完全够用。

如果你需要理解复杂的视觉内容（照片里的物体是什么、图表的数据趋势、UI 设计稿的布局评价），那还是需要真正的多模态模型。项目架构支持替换——把 `image-ocr.sh` 换成调用 Claude Vision / GPT-4V API 的脚本就行，其他部分不用改。如果你已经有 Claude API Key，这是个很简单的升级。

语音同理：飞书妙记只做转写，不会理解语气、情绪、环境音。要这些的话，换成多模态模型处理音频即可。

## 能做什么

- 飞书 Bot 收文字消息，Claude Code 自动处理并回复
- 发语音，自动转文字再处理
- 发图片，自动 OCR 提取文字再处理（支持 100+ 种语言，中英日韩等常用语言识别效果好）
- 让 Claude Code 读飞书文档、搜通讯录、发消息
- 断网后监听自动重连
- 妙记管理

## 为什么不用服务器

同类项目大多需要公网服务器接收飞书 Webhook。这套方案用 lark-cli 的 WebSocket 长连接，Bot 直连飞书服务器。没有公网 IP、没有端口映射、没有 HTTPS 证书。

## 怎么搭

把这段话发给 Claude Code，它会帮你全部搞定：

> 帮我搭建飞书 Claude Code 桥接，仓库在 https://github.com/blackcat-sub/feishu-claude-bridge

前置依赖安装：

```bash
./setup.sh    # 自动装 tesseract OCR + lark-cli，已装过的跳过
```

setup.sh 会自动处理：检测 tesseract、自动跳过已安装的组件。tesseract-lang 包含所有语言包（约 650MB），如果只识别中英文可以只保留所需语言包。

Claude Code 会自动完成的步骤：装 lark-cli → 创建飞书应用 → 弹浏览器让你授权 → 配白名单 → 下载脚本 → 启动监听。中途需要去飞书后台加权限时会告诉你。

## 文件说明

| 文件 | 用途 |
|------|------|
| `monitor.sh` | 消息监听，断网自动重连，全消息类型支持 |
| `feishu-stt.sh` | 语音转文字，传入 message_id 返回文本 |
| `image-ocr.sh` | 图片 OCR，传入 message_id 返回图中文字 |
| `cleanup-minutes.sh` | 妙记管理，`--open` 打开管理页清理 |
| `setup.sh` | 一键安装依赖（tesseract + lark-cli）|
| `settings.example.json` | Claude Code 权限白名单模板 |

## 语音和图片怎么处理

纯文本模型不能听语音、不能看图。两个免费路径绕开：

**语音 → 飞书妙记**（飞书自带）。下载语音 → 生成妙记 → 取逐字稿 → 删音频。不调第三方 ASR API，不花钱。

**图片 → tesseract OCR**（开源，本地离线）。下载图片 → OCR 提取文字 → 删本地文件。不调云 API，不花钱。

都处理完自动清理，不占空间。

## 注意

- 电脑锁屏可以，不能休眠
- 语音转写会产生妙记通知，可在飞书设置里关掉
- 想升级多模态？把 `image-ocr.sh` 替换为调 Claude Vision API 的脚本即可，其他不变
